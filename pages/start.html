Getting Started with Vitte (Enterprise-Grade)

Vitte blends Python’s clarity, Rust’s safety, and C/ASM control. The goal: ship fast, with correctness and performance by default.

Contents

1) Prerequisites

  • OS — Linux x86_64/aarch64, macOS (arm64/x64), Windows 10+ (x64).
  • Disk — 300–800 MB (toolchain + caches).
  • Toolchains —
    • Linux: clang ou gcc, make, headers.
    • macOS: Xcode CLT (xcode-select --install).
    • Windows: MSVC Build Tools ou LLVM; PowerShell récent.

2) Install the Toolchain

2.1 Unix one-liner

curl -fsSL https://get.vitte.dev/install.sh | sh

2.2 Windows (PowerShell)

iwr https://get.vitte.dev/install.ps1 -UseBasicParsing | iex

2.3 Package managers (community)

  • Homebrew: brew install vitte
  • Nix: nix-env -iA nixpkgs.vitte
  • Scoop (Win): scoop install vitte
  • Arch AUR: yay -S vitte

2.4 Manual

  1. Download from vitte.dev/downloads
  2. Extract, add bin/ to PATH

3) Corporate / Proxy / Air-gapped

3.1 Proxy env

export HTTP_PROXY=http://proxy.local:3128
export HTTPS_PROXY=http://proxy.local:3128
vitte self update

3.2 Private CA

export SSL_CERT_FILE=/etc/ssl/certs/corporate-ca.pem

3.3 Air-gapped

  1. Online: vitpkg vendor --all (lock + vendor deps)
  2. Transférer le dossier + toolchain offline
  3. Build: VITTE_OFFLINE=1 vitte build --release

5) Nix Flakes

# flake.nix (excerpt)
{
  description = "Vitte app";
  outputs = { self, nixpkgs, ... }:
  let pkgs = nixpkgs.legacyPackages.x86_64-linux;
  in {
    devShells.x86_64-linux.default = pkgs.mkShell {
      buildInputs = [ pkgs.vitte pkgs.clang ];
    };
  };
}

6) PATH, Shell & Completion

# PATH (Bash/Zsh) — adapt to your shell
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc && source ~/.bashrc

# Enable completion (if offered by your install)
vitte completions bash | sudo tee /etc/bash_completion.d/vitte
vitte completions zsh  | sudo tee /usr/local/share/zsh/site-functions/_vitte

13) Run, Profiles & Release

vitte run                 # dev profile (fast compile)
vitte build --release     # O3/LTO as configured
vitte build --profile perf

Tip: For CLI apps, prefer thin LTO and codegen-units=1 in vitbuild.toml for maximum perf.

14) Configuration Files

14.1 vitpkg.toml (manifest)

[package]
name = "hello"
version = "0.1.0"
edition = "2025"
license = "MIT"

[dependencies]
serde = "^1.0"

[features]
default = []
json = ["serde"]

[profile.dev]
opt-level = 0
debug = true
overflow-checks = true

[profile.release]
opt-level = 3
lto = "thin"
strip = "symbols"
codegen-units = 1
panic = "abort"

14.2 vitbuild.toml (build)

[project]
workspace = false

[target."x86_64-unknown-linux-gnu"]
linker = "clang"

[target."aarch64-apple-darwin"]
sdk = "macos"

[profile.perf]
opt-level = 3
lto = "fat"
codegen-units = 1

[scripts]
gen = "scripts/generate_assets.sh"

[hooks]
postbuild = "vitdoc build --theme clean"

[env]
VITTE_LOG = "info"

20) Debugging, Logs & Structured Logging

# Symbols
vitte build --debug

# gdb / lldb
gdb ./target/debug/hello
lldb ./target/debug/hello

# Verbose logs
VITTE_LOG=debug vitte run

Structured logging (JSON):

VITTE_LOG_FORMAT=json vitte run

21) Performance: Bench, Profile, Trace

# Bench
vitbench run --filter "json_*"

# Profile + flamegraph
vitprof record -- ./target/release/hello
vitprof flamegraph --open

# Quick trace (conceptual)
VITTE_TRACE=on vitte run

Perf tips: enable LTO (thin/fat), set codegen-units=1, avoid needless heap allocs, use iterators & slices, prefer Result over exceptions.

24) CI Quickstarts

24.1 GitHub Actions

name: ci
on: [push, pull_request]
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: curl -fsSL https://get.vitte.dev/install.sh | sh
      - run: vitte build --release
      - run: vitte test

24.2 GitLab CI (cache + artifacts)

stages: [build, test]
cache: { key: "$CI_COMMIT_REF_SLUG", paths: [".vitte/"] }
build:
  stage: build
  image: alpine:latest
  script:
    - wget -qO- https://get.vitte.dev/install.sh | sh
    - vitte build --release
  artifacts: { paths: ["target/release/"], expire_in: 1 week }
test:
  stage: test
  script: [ "vitte test" ]
