System Requirements & Supported Targets

Minimum

  • CPU: x86_64 or aarch64
  • RAM: 2 GB (8 GB recommended for large builds)
  • Disk: 500 MB (toolchain) + build artifacts
  • OS: Linux (glibc), macOS 12+, Windows 10/11 x64

Common targets (triples)

  • x86_64-unknown-linux-gnu, aarch64-unknown-linux-gnu
  • aarch64-apple-darwin, x86_64-apple-darwin
  • x86_64-pc-windows-msvc

Release Channels

Choose a channel:

  • stable — default, thoroughly tested.
  • nightly — latest features, may break.
  • LTS — long-term support, slow-moving.

Install script supports flags (examples):

# Unix-like
curl -fsSL https://vitte-lang.org/install.sh | sh -s -- --channel stable --prefix "$HOME/.local"

# Windows PowerShell
iwr https://vitte-lang.org/install.ps1 -UseBasicParsing | iex
vitte self channel set nightly

Quick Install

Unix-like (Linux, macOS)

curl -fsSL https://vitte-lang.org/install.sh | sh

Windows (PowerShell)

iwr https://vitte-lang.org/install.ps1 -UseBasicParsing | iex

After installation, confirm:

vitte --version
vitte help

Installers and Packages

Choose an installer for your platform. File names are illustrative; use the latest release.

Linux

  • x86_64 tar.gz
  • aarch64 tar.gz

macOS

  • Apple Silicon installer
  • Intel installer
  • Universal tarball

Windows

  • x64 MSI
  • x64 ZIP

Manual install from archive:

  1. Extract the archive.
  2. Add the bin/ directory to your PATH.

Unattended / Silent Install

Windows (MSI)

msiexec /i vitte-{latest}-x86_64-pc-windows-msvc.msi /qn /norestart INSTALLDIR="C:\Vitte"

macOS (pkg)

sudo installer -pkg vitte-{latest}-aarch64-apple-darwin.pkg -target /

Linux (tarball)

tar -xzf vitte-{latest}-x86_64-unknown-linux-gnu.tar.gz -C /opt/vitte
echo 'export PATH=/opt/vitte/bin:$PATH' | sudo tee /etc/profile.d/vitte.sh

Verify Downloads

Each download has accompanying .sha256 and .asc files.

Linux/macOS

# checksum
shasum -a 256 vitte-{latest}-x86_64-unknown-linux-gnu.tar.gz
cat vitte-{latest}-x86_64-unknown-linux-gnu.tar.gz.sha256

# signature (requires gpg and the Vitte release key)
gpg --keyserver hkps://keys.openpgp.org --recv-keys <VITTE_PUBKEY_ID>
gpg --fingerprint <VITTE_PUBKEY_ID>   # verify the fingerprint from the official site
gpg --verify vitte-{latest}.tar.gz.asc vitte-{latest}.tar.gz

Windows

certutil -hashfile vitte-{latest}-x86_64-pc-windows-msvc.msi SHA256
type vitte-{latest}-x86_64-pc-windows-msvc.msi.sha256

PATH and Shell Setup

Linux/macOS

# Bash
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc && . ~/.bashrc

# Zsh
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc && . ~/.zshrc

# Fish
set -Ua fish_user_paths $HOME/.local/bin

Windows

# PowerShell (current user)
$UserBin = "$env:USERPROFILE\.vitte\bin"
[System.Environment]::SetEnvironmentVariable("Path", "$env:Path;$UserBin", "User")

Update, Pinning, and Uninstall

# Update to latest
vitte self update

# Pin a specific version
vitte self pin 1.4.3

# Switch channel (stable/nightly/lts)
vitte self channel set nightly

# Uninstall (script-based installs)
vitte self uninstall

Enterprise / Proxy / Mirrors

Corporate networks often require proxies and custom CA bundles.

# Proxy
export HTTP_PROXY=http://proxy.local:3128
export HTTPS_PROXY=http://proxy.local:3128

# Custom CA
export SSL_CERT_FILE=/etc/ssl/certs/corporate-ca.pem

# Official mirror override (if provided)
export VITTE_MIRROR=https://mirror.corp.local/vitte

# Disable auto-update
export VITTE_NO_UPDATE=1

Offline / Air-gapped

  1. On a connected machine: vitpkg vendor --all in your project to vendor dependencies.
  2. Download the toolchain archives and signatures for your target(s).
  3. Transfer the toolchain and the project (including vendor/) to the offline environment.
  4. Install locally from archives and set offline mode:
VITTE_OFFLINE=1 vitte build --release

Troubleshooting

Command not found after install

Add the installation bin/ directory to your PATH and restart your shell. See PATH setup.

Permission denied when running installer

Make the script executable (chmod +x), or run via the shell as shown in the quick install.

macOS: “cannot be opened because it is from an unidentified developer”

Use the signed installer when possible. If using a tarball, you may need to clear quarantine: xattr -dr com.apple.quarantine /path/to/vitte.

Windows SmartScreen blocked

Use the MSI installer. If blocked, verify the signature and choose “Run anyway” only if verification succeeds.

Linker or SDK errors

Install a system toolchain (clang/lld or MSVC). Configure the linker in vitbuild.toml if needed. Ensure platform SDKs are present.

Behind a proxy or TLS interception

Set HTTP_PROXY, HTTPS_PROXY, and SSL_CERT_FILE to your corporate CA bundle. See Enterprise.

Antivirus slows builds on Windows

Exclude the build cache directories (e.g., .vitte/, target/) from real-time scanning.

Checksums and Signatures

Downloads include checksum and signature files alongside the artifacts:

  • vitte-{version}-<target>.tar.gz.sha256 and .asc
  • vitte-{version}-<target>.pkg.sha256 and .asc
  • vitte-{version}-<target>.msi.sha256 and .asc

Only install artifacts that pass checksum and signature verification. Always verify the PGP key fingerprint from the official website or an independent channel before trusting signatures.
